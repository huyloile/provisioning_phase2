Introduction

The aim of this tool is to provide as much information as possible about the customer's navigation, in order to cover all interrogations a customer may have. To achieve this, the tool allows to recreate as accurately as possible the customer's session.

Overview

Pages seen by a customer are saved as URLs. This implies that session-dependent dynamic elements are not stored and cannot be displayed by this tool. Dynamic elements are replaced by explicit terms.USSD sessions are saved on a 15 days sliding period, thus no information is available after.

Sessions search procedures

Once a CSR has logged in, the home page allows to :

  • Identify a customer

  • Select a time period

Customer identification

Customer identification is done according to his phone number (MSISDN) or to his IMSI number. Choice of the criterion is made possible by using radio buttons and drop down menu for periods

Examples of MSISDN inputs:

  • National format: 0678998877

  • International format: +33678998877

IMSI identification is done by entering the 15 digits.

NB : Customer IMSI is obligatorily known by the platform; this is not the case of the MSISDN. This is why IMSI is the preferred criterion.

Choice of period

This criterion allows to select only sessions that occurred during a specified period. Selected boundary days are included in the selection.

Search result

Once previous criterion have been selected, the CSR confirms his selection by clicking "Select" and the search result is displayed as described in the Section called USSD Customer card and the Section called Sessions list

USSD Customer card

This section aims at recalling customer details and is divided into two parts:

Customer summary

This summary recalls customer information. All the items may not appear depending on the platform configuration :

  • IMEI

  • IMSI

  • MSISDN (phone number in international format)

  • Subscription - the commercial profile as it appears in database (prepaid, postpaid, corporate...), if available

  • Language - a two letters code (en, fr...)

  • Date of the last connection to Cellcube (when available)

  • Additional specific information that is not available in a generic way

A 'NULL' value means that this information is not available. This may be normal for IMEI and language: default platform values are used in this case

Customer details

This section details the customer's terminal, its USSD navigation capabilities (if known) and other service values:

If you belong to the 'update' group, you can change values like the language, handset. Several kinds of editors are available depending on the running configuration. Main editors include :

  • Standard input: type the new value and click the update button next to the field

  • Standard list choice: select a value in the list and click the update button next to the field

  • Time editor: allows to edit a date/time. Format is DD-MM-YYYY HH:MM:SS

  • Handset Editor : allows to select the handset features by matching a requested terminal in the terminal database. The special keyword 'any' can be used to not filter on a given feature.

    Note : In the database, some terminals are set to use the platform default values for some of their capabilities. The 'any' value is the only way to grab them.

  • Handset Reset : delete current IMEI from database. Upon next connection, the customer will use default terminal capabilities

  • Handset IMEI : allows to specify an IMEI. Enter at least 6 digits.

  • Handset Choose IMEI : allows to select a kind of mobile depending on those that are defined in the platform configuration (e.g. : "182 characters" would match an IMEI with that number of characters)

Sessions list

Description

This list contains all the customer sessions during the selected period. They are sorted from the most recent to the oldest.

The following information is attached to each session :

  • Connection date

  • Connection time

  • First used service code

  • Session duration

  • Navigation phase that was used

  • Session cost

  • Phone/IMSI number used

A Maximum number of 15 sessions can be displayed on this page. If there are more sessions to print, they can be accessed using the "previous sessions" link described in the Section called Previous sessions.

For each session, it is possible to get a precise description of pages seen by the customer. For this, click on "details" and see the Section called Session description.

Next sessions

This link allows to select sessions that have been done after the most recent session displayed on screen.

Previous sessions

This link allows to select sessions that have been done before the oldest session displayed on screen.

Session description

This page is composed of a session summary displaying important information :

  • Connection date

  • Connection time

  • USSD Phase

    • 2 means that browsing is made using direct numbers : link 1 will be available by sending "1"

    • 1 means that browsing is made by redialing an USSD code : link 1 in service #123# would be reachable by dialing #12*1#

    • 1.5 means that browsing will be forced to use phase 1 style, even if the handset seems to handle USSD phase 2 (this can be a workaround for some handsets that do not support correctly phase 2)

  • Session cost

  • Session duration

  • Source identifier for the session : MSISDN or IMSI

A second part allows to see the complete navigation done by the customer, on four columns :

  • The first one indicates what the customer typed

  • The second column indicates how the USSD platform has interpreted the user input (if this interpretation is not the same as the customer's input)

  • The third column shows a brief summary for each visited page, or its URL. This is also a button that can be used to display the page content in the fourth column, as it has been sent to the customer (without some real dynamic content)

  • The fourth column displays the USSD page content if the user has clicked on the URL button or if the Cellcare application is configured to directly show sessions details. In this case, the user does not have to click on the button.

If the user browsed some taxed services/pages, some columns are added to describe their cost. Furthermore, a help about the different tariffs is displayed at the bottom of the page.

Data interpretation

Introduction

As we saw earlier, the tool recreates the customer's navigation using URL interpretation. As a consequence, some information may be modified or unavailable when recreating pages. This paragraph aims to explain a few re-interpretations as they may occur in the customer input or in the displayed page.

User input

The interpretation column allows to know how the platform has interpreted the input done by the customer. There are two kinds of interpretations :

  • First one is a Phase 1 navigation interpretation:

    A Phase 1 customer in the #123# service (for example), wishing to access link "1" will have to type on his handset "#12*1#"

    Example 1. Phase 1 interpretation

    "12*abcdef2" <=> "abcdef2"

  • The second kind of interpretation comes from the data itself that can be differently interpreted :

    Example 2. Conversion of a letter character into a numerical character

    "a" <=> "2";"d"<=>"3" ...

    Example 3. Conversion of a set of letter characters into a numerical set :

    "ad" <=> "2" then "3" or "23" <=> "2" then "3"

    Example 4. Conversion of characters into a platform-specific term

    "0" <=> "back" which means back to previous page

    "A" or "a" <=> "back"

    Ask your administrator what are the current platform-specific terms.

Pages interpretation

Sent pages/Browsed pages

Two kinds of pages can be displayed on screen:

  • Pages really sent to the customer : they are displayed with a gray background

  • Pages that are interpreted by the navigation engine to redirect the user to another page : (e.g. : Interpreted pages when using shortcuts #123*04#).

    These are displayed with a red background

Dynamic elements

Some pages contain some dynamic elements (balance, number of SMS...) that cannot be retrieved : these elements are replaced by red explicit terms.

These terms are not displayed on the terminal but they help the user to understand what was displayed on the customer's handset for real.

Some dynamic contents can not be replaced. In this case, "Missing elements"(in orange) is printed instead.

(NB : The term "Missing elements" means that one or several elements are not printed because they have been deleted or modified since the session time).

Updated pages

If a page has been deleted since the customer session, it will not be available for the session replay. The page URL will be displayed on a red background

If a page has been modified since the customer session, latest version will be displayed during the reply. As a consequence, browsing information may be inaccurate, for example if a link has been deleted from the page, the customer may have browsed the menu by following a 'virtual' link.

Split pages

During USSD navigation, some pages can contain a bigger number of characters than the customer's handset can support. These pages are cut into several parts according to the maximum number of characters the handset supports. These pages are split. The customer can access these split parts by following a special link "next" that will display the next part of the split page.

From the Cellcare tool point of view, this navigation is interpreted as follows :

  • The whole page is completely displayed in the first row.

  • Then, if the customer wishes to see the end of the page, his demand is interpreted as "Next page" and an empty cell appears in the "page" column if the customer followed the "next" link.

Note about language support

The customer has the ability to choose his language during navigation. Pages that are recreated in the customer care tool use the current language setting of the customer. This language parameter is session-independent. Thus, if the customer changes his language, the current setting in database will be used to recreate his session, which may not be the same as the one that was originally used during the USSD session.

Security

A login and a password are necessary to connect to the Customer care tool.

In order to authenticate yourself, your browser must accept cookies.

The authenticated session will end after an inactivity period.

Glossary

Session

A session contains all the actions made while the customer was browsing a service before it was interrupted, because of the user hangup or of the USSD server initiative.

Timeout

The timeout is a period during which the customer's session is still alive on the USSD server, waiting for a request from the customer. When this timeout occurs (e.g. : after 2 minutes), the session is closed on the USSD server.

IMEI
(International Mobile Equipment Identity)

The "id number" of a handset

IMSI
(International Mobile Subscriber Identity)

MSISDN
(Mobile Subscriber ISDN number)

UCS
(Universal Character Set)

Set of characters used to display some text other than simple ascii. Example : Greek, Arabic, Chinese...

In the USSD context, UCS pages can not exceed more than 80 characters.
